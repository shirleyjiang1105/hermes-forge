# Research Positioning & Paradigm Articulation

## When to Use

User is a researcher (especially resource-constrained: solo, early-career, multi-role) who needs to:
- Map multiple research lines driven by different collaborators
- Find their OWN unique research lane vs. what collaborators require
- Articulate a one-sentence research paradigm
- Choose which "申报类别" to lead with in grant applications
- Honestly assess competitors (not pretend no one else is in the space)
- Write a positioning narrative that shows coherence across seemingly scattered projects
- Prepare the strategic narrative for 导师制/申报表/项目申报书

Triggers: "我的研究范式是什么", "我跟别人有什么不同", "这个方向有人做吗", "申报类别怎么选", "我的几条线怎么串", "我在这个领域的位置是什么", "五年计划", "科研路线图", "帮我规划一下", "统领我的几个模块", "我的研究应该怎么做", or any mention of needing a multi-year roadmap, research identity, or portfolio structur

## Core Framework

### Step 1: Map ALL Research Lines (Brutal Honesty)

Draw every active line. Label each as **theirs** (collaborator-driven) or **yours** (self-initiated).

```
Collaborator A ← required → Line X
Collaborator B ← required → Line Y
                             ↑
                        these are SURVIVAL, not identity
─────────────────────────────────────────────
You          ← driven by → Line Z
                             ↑
                        this is your DIRECTION
```

Ask explicitly: "Which of these did YOU choose?" If all lines are collaborator-driven, the user may not have found their own lane yet. That's the first problem to solve.

### Step 2: Find the Cross-Disciplinary Wedge

The user's unique position is almost always at the intersection of their **institutional identity** and their **methodological skillset**.

Pattern:
- "I'm at a medical imaging college but I study classroom learning" → the wedge is "medical signal analysis applied to education"
- "I have EEG skills from clinical work but I teach undergraduate courses" → the wedge is "clinical-grade cognitive assessment in real classrooms"

The wedge exists where:
1. Competitor Group A (pure researchers) lacks the institutional access or teaching experience
2. Competitor Group B (pure educators) lacks the technical/methodological skills
3. The user has BOTH

### Step 3: Name the Competitors Honestly

NEVER say "this space is empty." It's not. Name the actual competitors:

| Competitor Group | Example | What They Have That You Don't | What You Have That They Don't |
|-----------------|---------|------------------------------|------------------------------|
| National lab researchers | 左西年/北师大 | Teams, cohorts, top journals | Classroom access, teaching practice |
| Ed school researchers | 各大师范大学教育学院 | Established programs, grants | Medical imaging background, clinical signal analysis |
| Clinical researchers | 院长/肺科合作者 | Clinical data, patient cohorts | Interest in education, classroom deployment |

The goal is NOT to claim superiority — it's to find the DIFFERENCE that gives you a defensible niche.

### Step 4: Articulate the Research Paradigm

A research paradigm is a one-sentence formula:

> **以[工具/方法]为手段，将[来源领域的分析范式]迁移至[目标场景]，通过[核心动作]来解决[核心问题]。**

Example from this session:
> "以便携EEG为工具，将医学影像信号分析范式迁移至真实课堂场景，通过外化学习者的认知过程来评估和优化教学质量。"

The paradigm must be:
- Specific enough to distinguish from competitors
- Broad enough to encompass multiple projects
- Rooted in what the user ACTUALLY does, not what they wish they did

### Step 5: Choose the Application Narrative

For Chinese academic forms (申报表, 导师制, 项目申报):

**Choosing "申报类别"**: Map all options against actual assets, pick the strongest relative position.

| Category | When to Choose |
|----------|---------------|
| 高水平学术论著 | User has 3+ SCI papers, even if none are top-tier |
| 省部级项目 | User has 1+ funded provincial/ministerial projects |
| 科技奖项 | Only if user has won a named award |
| Others | Only if user has concrete, verifiable outputs |

Honest framing: "You have 5 SCI papers, all in Q2-Q3. This isn't strong in absolute terms, but it's your strongest relative to other categories. Lead with it."

**Writing the positioning narrative**:
1. Lead with the user's OWN direction, not the collaborator-driven lines
2. Frame collaborator lines as "methodological foundation" supporting the main direction
3. Explicitly name the fit with the mentor/advisor's expertise as METHODOLOGY TRANSFER, not topic overlap
4. Don't overclaim — "进校不到2年的讲师有5篇SCI+1个省部级在研" is honest and sufficient

## Key Principles

- **Survival lines ≠ identity lines**: Collaborator-required projects are survival. The user needs a separate identity line.
- **Don't pretend there's no competition**: Always name the actual competitors and differentiate honestly.
- **The wedge is in the intersection**: Institutional identity × methodological skillset = unique position.
- **Resource constraints are design parameters, not excuses**: A solo researcher's paradigm should be designed for one person's throughput.
- **Narrative coherence > publication count**: A reviewer who sees 10 scattered projects will forget all of them. A reviewer who sees one clear paradigm across 3 projects will remember it.

### Step 6: Build the Multi-Year Staged Roadmap

After positioning, build a concrete roadmap. This bridges "who am I" to "what do I actually do each year."

**a) Layer the portfolio into a tiered structure**:

```
主攻轴 → core output projects (2-3 max)
方法根基 → capability-building, no standalone output
合作养线 → minimum-viable maintenance for collaborators
教学服务 → collateral that supports promotion dossiers
```

Each project gets assigned to exactly one tier. Tiers 2-4 serve Tier 1; Tier 1 delivers the narrative.

**b) Map to a methodological framework**:

Use an established paradigm as scaffolding. Example: 郁金泰's "Meta聚合→分化细做→回路加速":
- Phase 1 (Meta聚合): Systematic review → map the evidence landscape → identify gaps
- Phase 2 (分化细做): Deep-dive into the highest-leverage gap → one focused empirical contribution
- Phase 3 (回路加速): Results feed back into the map; extend to adjacent domains
- Phase 4 (工具化/闭环): Package as a reusable system or tool

The specific framework depends on the field, but the pattern is: **draw the map → dig one hole deep → connect the dots → make it usable.**

**c) Design for real resource constraints**:

- Solo researcher + one device? Design experiments around sequential (not simultaneous) data collection.
- No self-built cohort? The classroom IS the cohort — every semester is a natural longitudinal wave.
- Two collaborator ecosystems? Run their lines at minimum maintenance speed; invest in your own.
- Never compare output volume to well-resourced competitors. Compare positioning, not throughput.

**d) Produce a concrete timeline**:

| Year | Core Output | Capability Feed | Maintenance |
|------|------------|-----------------|-------------|
| Y1 | Meta + pilot | — | Collaborator line A |
| Y2 | First empirical paper | Method skills from Tier 2 | Collaborator line B starts |
| ... | ... | ... | ... |

Each row must have exactly ONE core output. If a year has three, it's not a plan — it's a wishlist.

**e) Save the plan for persistence**:

Write the full roadmap to a markdown file (e.g., `~/research-mgmt/master-plan.md`) and compress the key structure into persistent memory. The file is the canonical reference; memory is the always-available summary.

**Key principles for roadmaps**:
- **One core output per year, not three.** A solo researcher cannot deliver three papers, two grants, and a tool in one year.
- **Survival lines are budgeted at 10%, not 33%.** If a collaborator's project needs more than 10% of annual bandwidth, it IS the core output for that year — adjust the plan accordingly.
- **The classroom is infrastructure.** It's not a burden to be managed; it's the one asset pure researchers don't have.
- **Don't push the self-driven line to Year 3+ "when things settle down."** Things never settle down. Start it now, even if it's a tiny pilot.


## Common Pitfalls

- **Telling the user their space is "empty" or "no one does this"**: It's almost never true. Find the real competitors and differentiate.
- **Treating all research lines as equal**: Collaborator-driven lines and self-driven lines need different framing.
- **Overclaiming uniqueness**: "I'm the only one who..." is usually wrong. "I combine A and B in a way that most A-people and B-people don't" is usually right.
- **Comparing output volume to well-resourced competitors**: A solo lecturer cannot match a lab with 10 PhD students. Compare positioning, not volume.
- **Skipping the honest self-assessment**: If the user has no first-author Q1 papers, say so. The positioning must be built on reality.
- **Ignoring institutional gatekeeping ("包干到户")**: In many Chinese universities, NSFC submission is NOT just about merit — the institution allocates quotas per school/department. A lecturer may need a dean-level sponsor to get a submission slot. Before planning the research strategy, ask: "Who controls the slots in your institution? Do you have their support? If not, what would they need to see to back you?" Internal political positioning is as important as external peer review positioning.

## Reference: Naturalistic EEG Literature Chain Positioning

When the user's paradigm involves EEG in real classrooms, explicitly place them in the naturalistic EEG citation lineage:

```
MoBI paradigm (Makeig 2010s)
    ↓
Classroom brain-to-brain synchrony (Dikker 2017, Curr Biol; Bevilacqua 2023)
    ↓
Portable EEG + education assessment (Antonenko, Babiloni, etc.)
    ↓
★ USER'S POSITION: Teacher-researcher dual identity + domain-specific cognitive load taxonomy + interpretable framework
```

The user's unique advantage in this chain: they are BOTH the instructor and the researcher. Unlike pure researchers who must negotiate access to others' classrooms, the user designs the curriculum = designs the experiment. This is the single most defensible differentiator against well-resourced competitors.

## Reference: Four-Type Cognitive Load Taxonomy for Classroom EEG

When designing classroom EEG experiments, avoid the generic "high vs low cognitive load" dichotomy. Instead, map the course content to a four-type taxonomy:

| Load Type | Classroom Trigger | EEG Signature |
|-----------|------------------|---------------|
| Perceptual load | Interface complexity, information density | Occipital alpha suppression, P300 |
| Working memory load | Multi-step procedures, retention + execution | Frontal theta power, theta/alpha ratio |
| Task-switching load | Cross-modal interaction (touch → voice → gesture) | Frontal alpha asymmetry, switching cost |
| Problem-space load | Open-ended design challenges with constraints | Frontal theta synchronization, frontoparietal connectivity |

This taxonomy solves three problems: (1) it's theoretically grounded rather than arbitrary difficulty levels, (2) each type has a distinct EEG marker making classification cleaner, (3) it maps naturally to real course content without artificial task design.

## Reference: "3 New + 2 Old" Publication Strategy for Grant Applications

When the user is pivoting from an old domain (e.g., clinical neuroimaging) to a new domain (e.g., educational neuroscience) for a grant application, the 代表作 selection must serve two audiences simultaneously: peer reviewers (who need to see domain competence) and internal gatekeepers (who need to see productivity). The "3新+2旧" formula:

- **3 new-domain papers**: Systematic review (proves domain mastery) + methodological paper (proves technical innovation) + empirical paper in the target setting (proves feasibility)
- **2 old-domain papers**: Best-journal paper (proves publication capability) + most methodologically relevant paper (proves skill transferability)

The resulting reviewer impression: "This applicant has systematic output in the new domain AND a solid track record from the old domain — not a speculative career pivot."

## Reference: Course-as-Experimental-Platform Pattern

When the user teaches their own course, treat it as infrastructure, not overhead. Advantages over collaborating with others' classrooms:

1. Curriculum design = experimental design (no negotiation with other instructors)
2. Every semester = one longitudinal wave (natural cohort, no recruitment)
3. Instructor presence eliminates experimenter effects (students behave naturally)
4. Pre-test/post-test design is inherent (start of semester → end of semester)
5. Course content maps to cognitive load types (no artificial tasks needed)

The formula: "One EEG device + one course you teach + one semester = one complete study." This is the throughput a solo researcher can actually sustain.

## Reference: 郁金泰 Paradigm as Scaffolding

When helping a Chinese academic researcher map their path, the 郁金泰 model (复旦大学华山医院, AD/PD) provides a proven scaffolding:

| Phase | What He Did | Generic Pattern |
|-------|------------|-----------------|
| Meta聚合 | 2020 AD循证预防国际指南 | Systematic review → map the evidence landscape |
| 分化细做 | sTREM2 → YWHAG → FAM171A2 | Deep-dive into highest-leverage gap |
| 回路加速 | 预防↔诊断↔治疗互喂 | Results feed back into the map |
| 工具化 | 药物筛选/临床转化 | Package as reusable system/tool |

Key: The 2020 guideline came BEFORE the big empirical papers. The map enables the digging. For a solo researcher: do the meta-analysis FIRST, then the experiment.

## Reference: Classroom EEG Experiment Design (Single-Device)

When the user's paradigm involves EEG-based cognitive assessment in real classrooms with only ONE device:

**Two-phase sequential design** (not simultaneous multimodal):

Phase 1 (EEG): Diagnose WHERE cognitive overload happens. Pre-post or within-task, small stratified sample. Question: "Which stage causes the biggest theta/alpha spike?"

Phase 2 (Eye-tracking): Explain WHY. Same task, same students, different session. Question: "Do overloaded students show different visual search patterns?"

Why separate: (1) No sync hardware needed, (2) Each phase answers a distinct question, (3) Narrative arc: "EEG found the problem → eye-tracking explained the mechanism," (4) Feasible for one person with one device.

Data layers (deep to shallow): EEG+eye (1 student/session) → post-task self-report (whole class) → video playback behavior (whole class) → task scores (whole class). The classroom IS the longitudinal cohort — every semester is a new wave.

## Reference: Multi-Database Search Strategy (PICOS to 5 Databases)

When the user needs to write search strategies for a systematic review or meta-analysis from a PICOS framework, map to each database's syntax:

| Database | Syntax | Key Difference |
|----------|--------|----------------|
| PubMed | [Mesh] + [tiab] | MeSH terms available; use NOT for exclusions |
| Web of Science | TS=(...) | Topic search; Refine by Article/Review |
| Scopus | TITLE-ABS-KEY(...) | AND NOT for exclusions; LIMIT-TO for filters |
| IEEE Xplore | "Abstract":term | Engineering/device focus; Journals + Conferences |
| CNKI | SU%=(...) | Chinese keywords; * for AND; + for OR |

Save search strategies to project directory as `search_strategies.md`. This is reusable for future meta-analyses.
